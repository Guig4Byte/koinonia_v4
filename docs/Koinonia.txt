# Koinonia — visão futura/legada

Este arquivo é inspiração histórica e futura. Ele **não governa o MVP atual**.

Para qualquer decisão sobre a base atual, use primeiro:

1. `docs/AGENT_BRIEFING.md`
2. `docs/PRODUCT.md`
3. `docs/GLOSSARY.md`
4. `docs/ARCHITECTURE.md`

## Como ler este arquivo

As ideias abaixo podem orientar evolução futura, mas não autorizam implementação automática. Se algo aqui conflitar com o MVP Lite, ignore este arquivo e siga os documentos oficiais acima.

## Estado atual que não é futuro

O MVP atual já possui:

- autenticação real simples por e-mail e senha;
- sessão em cookie `HttpOnly`;
- tema local no login e no app;
- visão por papel;
- busca de pessoa;
- presença, atenção, contato e cuidado.

Não trate esses itens como pendências futuras.

## Visão ampla

O Koinonia pode se tornar um sistema mais completo de cuidado pastoral para igrejas com células, combinando presença, vínculo, leitura de saúde dos grupos e ações de cuidado.

Possibilidades futuras:

- gestão simples de usuários e troca de senha;
- recuperação de acesso;
- convites controlados;
- relatórios mais profundos;
- acompanhamento formal para casos contínuos;
- área de membro;
- notificações;
- recursos de integração;
- analytics e indicadores históricos;
- mapas ou geolocalização;
- QR Code;
- playbooks de cuidado;
- políticas avançadas de privacidade e auditoria.

## Limite para o MVP atual

No MVP Lite atual, essas possibilidades **não entram por padrão**:

- cadastro público;
- CRM pastoral completo;
- task manager;
- SLA;
- funil;
- BI executivo;
- mapas;
- QR Code;
- geolocalização;
- notificações;
- formulários longos;
- acompanhamento formal;
- permissões granulares customizadas.

O MVP atual continua governado por:

```txt
Evento -> Presença -> Atenção -> Contato -> Cuidado
```

E por:

```txt
Líder registra.
Supervisor acompanha.
Pastor interpreta.
```

## Nota para agentes

Não use este arquivo como lista de requisitos. Use-o apenas para entender a ambição futura do produto depois de respeitar o recorte oficial do MVP.
