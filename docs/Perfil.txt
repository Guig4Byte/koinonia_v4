# Experiência — norte mobile/pastoral

Este arquivo orienta sensação de uso, densidade visual e tom. Ele não define escopo técnico nem substitui `PRODUCT.md`, `GLOSSARY.md` ou `ARCHITECTURE.md`.

## Persona emocional

Imagine Roberto, pastor, 52 anos, usando o celular à noite, cansado, sem querer “operar um sistema”. Ele quer saber rapidamente se alguém está escapando do cuidado e onde precisa prestar atenção.

O Koinonia deve funcionar como um termômetro de bolso, não como painel de controle.

## Princípio de experiência

```txt
Abrir, entender, respirar e agir.
```

A tela deve aliviar o peso pastoral, não gerar mais uma obrigação administrativa.

## Primeira leitura

A primeira tela deve responder em poucos segundos:

- está tudo razoavelmente bem?
- existe alguém que pede cuidado mais próximo?
- alguma célula está sem presença ou com presença baixa?
- preciso agir agora ou só acompanhar?

Evite telas iniciais com muitos cards, gráficos, tabelas ou linguagem executiva.

## Navegação mobile

A navegação deve caber no polegar e usar poucos destinos.

A estrutura atual trabalha com:

- `Visão`;
- `Pessoas` ou `Membros`, conforme o perfil;
- `Eventos`.

Não adicionar novas abas sem necessidade clara. Uma nova aba só entra se reduzir esforço real de cuidado.

## Busca

A busca é central, especialmente para pastor e supervisor.

Uso típico:

```txt
Alguém menciona uma pessoa no corredor -> usuário busca o nome -> abre pessoa -> entende contexto rápido.
```

Busca deve ser rápida, simples e contextual. No MVP atual, é busca de pessoa.

## Feed pastoral, não planilha

Listas devem parecer feed de cuidado, não tabela operacional.

Cada card deve mostrar:

- quem é a pessoa;
- contexto curto;
- motivo de atenção;
- status claro;
- CTA simples, normalmente `Abrir pessoa`.

Ações mais sensíveis pertencem ao detalhe da pessoa.

## O que evitar no mobile

Evite:

- tabelas com muitas colunas;
- formulários longos;
- gráficos complexos;
- filtros avançados prematuros;
- blocos repetidos de botões;
- linguagem de cobrança;
- “fila de tarefas” pastoral.

## Tom

A UI deve soar como cuidado e clareza:

- curto;
- humano;
- sem culpa;
- sem jargão corporativo;
- sem prometer mais do que a tela entrega.

Frase guia:

```txt
Mostre só o que ajuda alguém a cuidar melhor de alguém.
```
